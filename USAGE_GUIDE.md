## 📋 执行模式

### Backtest模式（推荐用于日常使用）

```bash
python run_asa.py --mode backtest --max-stocks 50 --max-factors 20
```

**特点：**
- 仅执行核心量化步骤
- 不包含数据更新
- 速度快，适合频繁测试
- 包含因子计算、验证、Alpha生成、策略回测、组合优化

### Standard模式

```bash
python run_asa.py --mode standard --max-stocks 100 --max-factors 20
```

**特点：**
- 完整的每日管线
- 包含数据更新步骤
- 适合日常运营
- 可能遇到退市股票问题

### Fast模式

```bash
python run_asa.py --mode fast --max-stocks 500 --max-factors 20
```

**特点：**
- 快速执行模式
- 跳过部分非核心步骤
- 适合快速验证

### Live模式

```bash
python run_asa.py --mode live --max-stocks 20 --max-factors 15
```

**特点：**
- 实盘交易模式
- 包含交易执行
- 需要配置交易接口

## 🔧 参数说明

| 参数 | 说明 | 默认值 | 推荐值 |
|------|------|--------|--------|
| `--mode` | 执行模式 | standard | backtest |

## 🛠️ 故障排除

### 常见问题

**Q: 执行超时怎么办？**
A: 减少max-stocks和max-factors参数

**Q: 质量门控警告严重吗？**
A: 小规模测试的警告是正常的，增加股票数量会改善

**Q: 如何查看详细日志？**
A: 检查logs目录下的日志文件

**Q: 交易日历警告需要处理吗？**
A: 不需要，系统会自动回退到内置日历

**Q: LLM审核不通过怎么办？**
A: 系统会自动诊断问题根源并修复根本问题，然后重新运行工作流

**Q: 如何手动修复根本问题？**
A: 运行 `python3 fix_root_cause.py <diagnoses_json>` 命令，根据诊断结果修复问题

**Q: 如何查看配置文件？**
A: 检查config目录下的factor_config.json、strategy_config.json、risk_config.json

### LLM审核和根本问题修复（v6.5.2）

**审核标准：**
- 能够盈利：年化收益≥5%，夏普比率≥0.5，胜率≥50%
- 风险可控：最大回撤≤-20%，波动率≤30%
- 可以操作：有明确的买入数量和价格，买入信号数量≥5个

**问题根源诊断：**
- 盈利能力问题 → 诊断是策略问题还是因子问题
- 风险控制问题 → 诊断是风险控制问题还是策略问题
- 可操作性问题 → 诊断是数据问题还是策略问题

**根本问题修复：**
- 因子问题 → 优化因子组合，提高因子质量
- 策略问题 → 优化选股逻辑，调整选股参数，添加止盈止损
- 风险控制问题 → 优化仓位管理，添加止损机制
- 数据问题 → 重新获取股票数据，确保数据完整
