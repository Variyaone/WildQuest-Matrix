## 📋 系统概述

### 核心问题

旧系统存在以下问题：
1. ❌ 指挥官做了太多执行工作，没有有效分配任务
2. ❌ 其他agent API使用率低（说明闲置）
3. ❌ 缺少任务池和看板机制

### 新系统目标

建立高效的自主任务系统：

1. ✅ **任务池机制** - agent可以自主领取任务
2. ✅ **看板机制** - 任务状态可视化（待执行、进行中、已完成）
3. ✅ **简化系统** - 减少信息熵，移除冗余文档
4. ✅ **自主工作机制** - agent可以自行查阅任务清单并执行

## 🏗️ 系统架构

### 核心文件

| 文件 | 用途 | 目标读者 |
|------|------|---------|
| `TASK_POOL.md` | 任务池，所有待执行任务清单 | Agent |

### 工作流程

```
指挥官添加任务
    ↓

Agent自主领取
    ↓
TASK_KANBAN.md（进行中）
    ↓
Agent执行任务
    ↓
TASK_KANBAN.md（已完成）
```

## 📝 指挥官操作指南

### 添加任务

编辑 `TASK_POOL.md`：

```markdown
### P0 - 紧急
- [ ] `[领取]` **任务标题**
  - 描述：任务详细描述
  - 要求：验收标准
  - 预计时间：X小时
  - 负责人：待领取
```

### 监控进度

查看 `TASK_KANBAN.md`：
- 统计各状态任务数量
- 查看"进行中"任务的进度
- 检查Agent工作分配

### 指导方针

1. **少做执行，多做指挥**
   - 添加任务到池
   - 设置优先级
   - 宏观把控进度

2. **信任Agent**
   - 让Agent自主决策
   - 只关注结果，不过度干预
   - 在关键时刻给予指导

3. **优化流程**
   - 定期检查系统效率
   - 收集Agent反馈
   - 持续改进机制

## 🚦 系统健康监控

### 需要关注的情况

🔴 **紧急**
- 超过48小时无任务完成
- 3个以上Agent僵尸化
- 关键任务超时

🟡 **需要注意**
- 任务池为空超过12小时
- 某个Agent7天无活动
- 完成率低于30%

🟢 **健康**
- 每日有任务完成
- Agent使用率 >70%
- 系统运行流畅

## 📚 Agent使用技巧

### 激活僵尸Agent

如果发现Agent僵尸化（无活动超过2天）：

1. 在任务池中添加适合该Agent的任务
2. 标记为P0或P1优先级
3. Agent会在心跳检查时发现并领取

### 分配特定任务

如果希望特定Agent处理某任务：

在任务池中添加备注：
```markdown
- [ ] `[领取]` **任务标题**
  - 描述：任务详细描述
  - 要求：验收标准
  - 预计时间：X小时
  - 负责人：建议由研究员处理
```

## 🚨 常见问题

### Q: 总是同一个agent领取任务怎么办？
A: 调整任务池，添加更多适合其他agent的任务

### Q: Agent任务质量不好怎么办？
A: 在任务要求中明确验收标准，完成后由评审官检查

### Q: Agent领取任务后不执行怎么办？
A: 检查Agent状态，如僵尸化需要重新激活

### Q: 指挥官是否还是要分配任务？
A: 可以，但只需在任务池中标记"建议由XXX处理"
