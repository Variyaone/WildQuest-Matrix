## 📋 阶段门控机制（Stage Gates）

每个阶段有明确的**完成标准**和**验收人**，必须全部满足才能进入下一阶段。

### 阶段门控矩阵

| 阶段 | 名称 | 完成标准 | 验收人 | 必通过 | 备注 |
|------|------|---------|--------|--------|------|
| Gate 0 | 项目启动 | ✅ 目标明确<br>✅ 团队组建 | 指挥官 | ✅ | 启动前 |
| Gate 1 | 数据分析完成 | ✅ 历史数据获取（≥30天）<br>✅ 波动率分析完成<br>✅ 适用性评估报告 | 研究员+评审官 | ✅ | 不得跳过 |
| Gate 2 | 回测完成 | ✅ 回测框架实现<br>✅ 策略回测运行<br>✅ 回测报告（收益率/回撤/夏普比率） | 架构师+评审官 | ✅ | **强制** |
| Gate 3 | 参数优化完成 | ✅ 多组参数测试<br>✅ 最优参数确定<br>✅ 风险评估矩阵 | 研究员+评审官 | ✅ | **强制** |
| Gate 4 | 代码审查通过 | ✅ 代码质量检查<br>✅ 逻辑正确性验证<br>✅ 风险识别完成 | 评审官 | ✅ | **必须** |
| Gate 5 | 小规模测试通过 | ✅ 模拟盘测试（1周）<br>✅ 盈亏追踪<br>✅ 异常处理验证 | 评审官+指挥官 | ✅ | **强制** |
| Gate 6 | 最终审批 | ✅ 风险评估完成<br>✅ 风控措施到位<br>✅ 备选方案准备 | 评审官 | ✅ | **最后一关** |
| Gate 7 | 生产部署 | ✅ 监控告警配置<br>✅ 熔断机制启用<br>✅ 人工确认 | 指挥官 | ✅ | 双重确认 |

## 🔴 强制检查点（不可跳过）

### 检查点1：回测必须有结果
```
必须满足：
✅ 回测跑通，无错误
✅ 有具体的数值结果：
   - 总收益率
   - 最大回撤
   - 夏普比率
   - 胜率
   - 盈亏比

❌ 不允许：跳过/仅有定性描述/说"可以试试"
```

### 检查点2：风险评估必须有矩阵
```
必须包含：
✅ 风险识别清单（市场/流动性/技术）
✅ 风险评级（高/中/低）
✅ 概率+影响评估
✅ 应对措施

❌ 不允许：无风险评估/仅有"注意风险"的表述
```

### 检查点3：评审官必须签字
```
必须从评审官获得明确答复：
✅ "通过" - 可以进入下一阶段
⚠️ "有条件通过" - 需要补充X后进入
❌ "不通过" - 必须修改后重新评审

❌ 不允许：不经评审直接进入下一阶段
```

### 检查点4：参数必须优化过
```
必须提供：
✅ 至少3组参数的回测对比
✅ 参数敏感度分析
✅ 最优参数选择理由

❌ 不允许：使用默认值/未验证参数
```

## 🚨 决策流程可视化

```
开始
   ↓
[Gate 0] 项目启动 ──────→ ❌ 目标不明：回退明确
   ↓ ✅
[Gate 1] 数据分析
   ├─ ✅ 波动率适中（1.5%-3%）
   ├─ ✅ 震荡市场（趋势<5%）
   ├─ ✅ 区间停留>70%
   └─ ↓
[Gate 2] 回测（CRITICAL）
   ├─ ❌ 回测报错/无结果 → 停止，修复后再试
   ├─ ❌ 收益率<预期 → 调整策略
   ├─ ❌ 回撤>20% → 调整参数
   └─ ✅ 回测指标达标 → 继续
   ↓
[Gate 3] 参数优化
   ├─ ✅ 多组参数对比
   └─ ✅ 最优参数确定
   ↓
[Gate 4] 代码审查
   ├─ ❌ 重大bug → 修改
   ├─ ❌ 缺少异常处理 → 补充
   └─ ✅ 评审官通过 → 继续
   ↓
[Gate 5] 小规模测试
   ├─ ❌ 异常频发 → 调试
   ├─ ❌ 实际波动偏离预期 → 重新回测
   └─ ✅ 测试稳定 → 继续
   ↓
[Gate 6] 最终审批（REVIEWER）
   ├─ ❌ 风险不可控 → 拒绝
   ├─ ❌ 风控措施不足 → 补充
   └─ ✅ 评审官签字 → 最后一关
   ↓
[Gate 7] 生产部署
   ├─ ✅ 监控告警配置
   ├─ ✅ 熔断机制启用
   └─ ⚠️ 人工确认（不可自动化）
   ↓
上线
```

## 📊 风险量化评估矩阵

每个决策必须评估风险：

| 风险类型 | 风险点 | 概率 | 影响 | 风险等级 | 应对措施 | 负责人 |
|---------|--------|------|------|---------|---------|--------|
| 市场风险 | 单边趋势突破策略区间 | 高 | 高 | 🔴 极高 | 设置止损+动态调整区间 | 评审官 |
| 流动性风险 | 订单无法成交 | 中 | 中 | 🟡 中 | 挂单+滑点限制 | 架构师 |
| 技术风险 | API故障 | 低 | 高 | 🟠 中-高 | 重连机制+人工接管 | 创作者 |
| 回测过拟合 | 历史表现≠未来表现 | 高 | 高 | 🔴 极高 | 样本外验证+多周期测试 | 架构师 |
