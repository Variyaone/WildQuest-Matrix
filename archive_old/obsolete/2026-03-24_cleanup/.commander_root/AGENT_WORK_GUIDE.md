## 📋 快速开始（3步）

### 第1步：检查是否有工作

```bash
# 步骤1.1：检查任务池
cat /Users/variya/.openclaw/workspace/.commander/TASK_POOL.md

# 步骤1.2：检查你的Agent状态（可选）
# 在TASK_KANBAN.md中查看你的任务分配情况
```

### 第2步：领取任务（如果有可用任务）

编辑 `TASK_POOL.md`：
- 在任务前添加 `[领取:你的Agent名称]`
- 例如：`- [x] [领取:研究员] **A股市场因子研究**`

### 第3步：更新看板

编辑 `TASK_KANBAN.md`：
- 将任务从"待执行"移至"进行中"
- 填写你的Agent名称和开始时间
- 开始执行任务

### 第4步：完成任务后

编辑 `TASK_KANBAN.md`：
- 将任务移至"已完成"
- 填写完成时间和结果

## 🎯 Agent职责

### 工作原则

1. **自主性** - 主动查看任务池，不等待分配
2. **优先级** - 优先处理P0和P1任务
3. **并发** - 同时任务不超过2个
4. **沟通** - 遇到阻塞及时更新状态
5. **质量** - 确保完成任务的质量

### 无任务时的行为

如果任务池为空或没有适合的任务：
1. 检查是否需要激活其他agent
2. 寻找可以优化或改进的工作
3. 主动询问指挥官是否有新任务
4. 不要 idle，保持学习或帮助其他agent

## 🚨 异常处理

### 任务阻塞
- 如果任务被阻塞（等待审核、缺少资源等），立即在 `TASK_KANBAN.md` 中标记为"暂停"并说明原因
- 同时更新 `TASK_POOL.md` 的任务状态

### 无法完成任务
- 如果发现任务超出能力范围，说明原因，将任务放回任务池
- 可在任务中添加备注："建议由XXX处理"

### 需要帮助
- 如果需要其他agent协作或指挥官支持，在任务描述中注明

## 💡 最佳实践

### 1. 定期检查（建议每2-4小时）
- 检查任务池是否有新任务
- 检查其他agent的状态
- 更新自己的任务进度

### 2. 保持同步
- 开始任务时立即更新看板
- 做完任务立即更新状态
- 遇到问题及时记录

### 3. 质量第一
- 不要只是完成任务，要确保质量
- 产出结果要清晰、可验证
- 必要时添加文档说明

### 4. 主动沟通
- 发现优化机会要主动提出
- 帮助其他agent解决问题
- 分享经验和教训
