# 📚 历史错误与教训清单

> **目的**: 避免重复犯错，持续改进

## 🔴 重大错误（已发生，必须杜绝）

### 1. 组织效率低下
- **错误**: 指挥官做执行工作，其他agent闲置
- **后果**: 资源浪费，效率极低
- **根因**: 分工不清晰，subagent缺少独立能力
- **改进**: ✅ 已创建SOUL.md，强化独立性
- **验证**: subagent独立完成任务率需>80%

### 2. AI幻觉泛滥
- **错误**: agent声称"完成"但无验证
- **后果**: 任务实际未完成，问题积累
- **根因**: 缺少强制验证机制
- **改进**: ✅ SOUL.md中强制验证要求
- **验证**: 所有声称"完成"的任务必须有验证命令和结果

### 3. 数据质量问题
- **错误**: mock_data.pkl不是真实数据，价格逻辑错误
- **后果**: 回测结果不可信
- **根因**: 数据源不可靠，缺少质量检查
- **改进**: 🔄 接入真实数据源
- **验证**: 数据质量检查脚本

### 4. OKX项目资源分散
- **错误**: 同时搞多个项目，资源分散
- **后果**: 都做不好
- **根因**: 战略不聚焦
- **改进**: ✅ OKX项目终止，聚焦A股
- **验证**: 资源100%集中到A股

### 5. 任务验证缺失
- **错误**: 代码跑不通、策略无回测、文档格式错误
- **后果**: 交付质量差
- **根因**: 无验证协议
- **改进**: ✅ 强制验证机制
- **验证**: critic必须用实际命令验证

## 🟡 流程问题（需要优化）

### 1. Cron任务冗余
- **问题**: 健康检查和超时处理分开，资源浪费
- **改进**: 🔄 合并为统一系统监控
- **验证**: cron任务数减少

### 2. 记录系统冗余
- **问题**: cron每日总结和memory重复
- **改进**: ✅ 已移除cron总结
- **验证**: memory是唯一记录源

### 3. auto_trigger.py僵尸进程堆积
- **问题**: auto_trigger.py是死循环脚本，但cron每5分钟启动一次，产生大量僵尸进程
- **后果**: 系统资源浪费，进程数量爆炸
- **根因**: 持续运行脚本与cron定时触发冲突
- **改进**: ✅ crontab改为`--once`模式（单次执行）
- **验证**: ps检查无僵尸进程累积

### 4. 缺少常态化优化
- **问题**: 出问题才改，不主动优化
- **改进**: 🔄 建立定期优化机制
- **验证**: 每周系统优化报告
