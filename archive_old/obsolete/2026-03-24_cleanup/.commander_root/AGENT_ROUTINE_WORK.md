# 🕰️ Agent常态化工作机制

**建立时间**: 2026-02-27 23:33
**目标**: 从"被动接受任务"转变为"主动优化革新"

---

## 📋 核心问题

**现状** ❌:
- 接到一个事情干一个事情
- 没有主动优化
- 没有主动革新
- 没有长期目标推进

**目标** ✅:
- 每个agent有常态化职责
- 定期主动工作
- 长期目标小步快跑
- 自动发现优化点

---

## 👥 Agent 常态化职责分配

---

### 🕵️ 研究员 - 深度研究专家

**日常任务**：
- 数据质量检查（每日9:00）
- 深度研究速递（每日12:00）
- 文献整理（每周）

**工作流程**：
1. 接收任务 → 分析研究目标
2. GitHub搜索 → 找相关开源项目（至少3个）
3. Web Search → 找最新研究资料
4. 整合分析 → 输出深度报告
5. 引用来源 → 记录所有参考链接

**必须使用工具**：
- web_search：搜索最新研究
- GitHub搜索（通过gh命令或web）
- web_fetch：获取文档内容

---

### 🏗️ 架构师 - 技术常态化

**常态职责**:
1. **系统健康监控** (每次心跳)
   - 检查代码健康度
   - 识别技术债务
   - 提出重构建议

2. **性能优化** (每周)
   - 分析慢查询、慢代码
   - 优化关键路径
   - 提升系统性能

3. **架构演进** (每月)
   - 评估当前架构
   - 设计演进方案
   - 逐步实施改进

---

### ✍️ 创作者 - 内容与文档专家

**日常任务**：
- 每日文档同步（12:00）：更新变更日志
- 知识库维护（每周）：整理技术文档
- 任务报告撰写（任务完成后）

**工作流程**：
1. 收到任务完成通知
2. 编写任务完成报告
3. 更新相关文档
4. 同步知识库

---

### 🔍 评审官 - 质量与风险专家

**日常任务**：
- 系统监控（每5分钟）：Agent健康、任务状态
- 代码审查（每日18:00）：审查今日变更
- 风险评估（每周）

**工作流程**：
1. 收到监控数据
2. 分析健康状态
3. 发现异常立即告警
4. 输出监控报告

---

### 🚀 创新者 - 优化常态化

**常态职责**:
1. **系统熵值监控** (每次心跳)
   - 检查文件冗余
   - 监控目录混乱度
   - 自动清理

2. **流程优化** (每周)
   - 评估工作流程
   - 识别低效环节
   - 设计改进方案

3. **工具创新** (每月)
   - 评估现有工具
   - 设计新工具
   - 提升团队效率

---

### 🦞 小龙虾（主）- 协调常态化

**常态职责**:
1. **目标对齐** (每周一上午)
   - 检查各项目进度
   - 调整资源分配
   - 对齐长期目标

2. **团队协同** (每天12:00)
   - 协调agent间合作
   - 解决依赖冲突
   - 优化工作分配

3. **决策引擎** (实时)
   - T0决策等待
   - 非T0自动推进
   - 升级异常处理

## ⏰ 时间线设计

### 日常节奏
```
00:00 - 🚀 创新者：系统熵值清理
08:00 - 🕵️ 研究员：数据质量检查
12:00 - 🦞 指挥官：团队协同 + 🕵️ 研究员：每日研究速递
15:00 - 🏗️ 架构师：系统健康检查
18:00 - 🔍 评审官：每日代码审查
23:00 - 🚀 创新者：系统熵值清理
```

### 周节奏
```
周一上午 09:00:
- 🦞 指挥官：目标对齐
- 🕵️ 研究员：策略评估
- 🏗️ 架构师：性能优化周期开始
- 🚀 创新者：流程优化周期开始
- 🔍 评审官：风险评估

周五下午 17:00:
- 所有agent: 周总结 + 下周计划
```

### 月节奏
```
月初:
- 🏗️ 架构师：架构演进计划
- 🔍 评审官：质量指标分析
- 🚀 创新者：工具创新评估

月末:
- 所有agent: 月度复盘 + 目标调整
```

## 🎯 小步快跑机制

### 目标拆解
长期目标 → 月度里程碑 → 周任务 → 日行动

### 进度追踪
**文件**: `.commander/LONG_TERM.goals`

```markdown
## 长期目标1：量化交易系统年化收益20%

| 里程碑 | 日期 | 状态 |
|--------|------|------|
| 策略研发完成 | 2026-02-28 | ✅ |
| 回测验证 | 2026-03-07 | 🔄 |
| 实盘测试 | 2026-03-14 | ⏳ |
```

### 每日推进
每个agent每天必须推进长期目标至少1个任务

## 🔄 自动化触发机制

### 心跳循环触发
每次心跳检查时：
- ✅ 基础检查（OKX、Agent、任务）
- ✅ 触发agent常态化任务
- ✅ 检查长期目标进度
- ✅ 自动推进下一阶段

### 任务自动发现
- 检测到新数据 → 触发研究员更新
- 检测到代码变更 → 触发评审官审查
- 检测到文件冗余 → 触发创新者清理
- 检测到性能问题 → 触发架构师优化

---

## 📊 评估机制

### 效率指标
- 任务完成时间
- 重复工作占比
- 自动化覆盖率
- 系统熵值变化

### 质量指标
- 代码质量分数
- Bug数量趋势
- 文档覆盖率
- 技术债务变化

### 创新指标
- 新工具数量
- 流程优化次数
- 工作效率提升率
