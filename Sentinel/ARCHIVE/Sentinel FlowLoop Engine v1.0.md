# 🔄 Sentinel FlowLoop 引擎

> **版本**: v1.0 - 极简三文件版
> **核心理念**: 极简三文件 + 自动闭环 + 智能升级
> **最后更新**: 2026-03-12

---

## 一、核心设计原则

### 1.1 极简主义
- 三文件驱动：`TODO.md` | `DONE.md` | `T0.md`
- 所有Agent共享访问，无状态冲突
- 一次心跳，一个循环，全部闭环

### 1.2 T0严格定义
**只有以下情况才需要用户确认**：
- 🚨 系统安全风险（数据泄露、权限越界、恶意代码）
- 💰 真实资金损失风险（交易、转账、支付）
- 🗑️ 不可逆数据操作（删除、覆盖、格式化）
- 📤 对外发送消息（可能影响用户声誉）

**其他一切事务，Agent自主决策执行！**

### 1.3 自动升级机制
```
普通任务失败3次 → 自动升级为T0 → 记录失败原因 → 等待用户决策
```


```
│  │ ├─ 检查T0.md → 有待确认项 → 通知用户                   │  │
│  │ └─ 确认各项目主目标（首次心跳时）                       │  │
│  └──────────────────────────────────────────────────────┘  │
│                           │                                  │
│                           ▼                                  │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ 步骤2: 发散探索                                        │  │
│  │ ├─ 是否有更深入的想法？ → 写入TODO                     │  │
│  │ ├─ 是否有可改进的点？ → 写入TODO                       │  │
│  │ └─ T0任务尝试替代方案（不触碰T0边界）→ 写入TODO         │  │
│  └──────────────────────────────────────────────────────┘  │
│                           │                                  │
│                           ▼                                  │
│  ┌──────────────────────────────────────────────────────┐  │

│  │ ├─ 压缩并生成摘要 → 追加到DONE.md                      │  │
│  │ └─ 修剪冗余内容，保持主题聚焦                          │  │
│  └──────────────────────────────────────────────────────┘  │
│                           │                                  │
│                           ▼                                  │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ 步骤4: 任务执行                                        │  │
│  │ ├─ 按优先级执行TODO中的任务                            │  │

│  │ ├─ 优先级：上下文 > T0文档 > DONE记录                  │  │
│  │ ├─ 寻找可改进/补充的任务 → 写入TODO                    │  │
│  │ └─ 返回步骤1                                           │  │
```

## 四、详细执行规则

### 4.1 任务分解规则

**触发条件**：任务预计耗时 > 30分钟 或 包含多个子目标

**分解流程**：
```
原任务: TASK-001 实现用户认证系统
    │
    ├─ 分解为:
    │   ├─ TASK-001a: 设计认证流程
    │   ├─ TASK-001b: 实现登录接口
    │   ├─ TASK-001c: 实现权限校验
    │   └─ TASK-001d: 编写测试用例
    │
    └─ 立即写入TODO.md，标注父任务关系
```

### 4.2 失败升级规则

```
任务执行流程:
    │
    ├─ 第1次失败 → 记录原因 → 重试
    │
    ├─ 第2次失败 → 记录原因 → 尝试替代方案
    │
    ├─ 第3次失败 → 升级为T0
    │   ├─ 写入T0.md
    │   ├─ 记录3次失败原因
    │   ├─ 提供可选解决方案
    │   └─ 通知用户
    │
    └─ 等待用户决策后继续
```

### 4.3 空窗期探索优先级

```
TODO为空时的探索顺序:

1️⃣ 上下文延续（权重50%）
   └─ 基于当前对话，提出逻辑下一步

2️⃣ T0文档启发（权重30%）
   └─ 分析T0.md中的历史问题，寻找改进点

3️⃣ DONE记录挖掘（权重20%）
   └─ 扫描已完成事项，寻找可优化/文档化/自动化的点
```

### 4.4 上下文压缩规则

**触发条件**：
- 对话轮数 > 50轮
- Token数 > 10,000
- 主题开始发散

**压缩流程**：
```
1. 提取核心决策和结论
2. 生成摘要（格式如下）
3. 追加到DONE.md
4. 清理冗余对话

摘要格式:
## CTX-YYYYMMDD-XXX
- 核心内容：...
- 关键决策：...
- 待办事项：[TASK-XXX]
- 相关上下文：[CTX-YYY]
```

## 五、Agent协作机制

### 5.1 角色分工

| Agent | 职责 | 可访问文件 |
|-------|------|-----------|
| **main** | 协调、分配、决策 | 全部 |
| **researcher** | 研究、分析 | TODO/DONE |
| **architect** | 代码、执行 | TODO/DONE |
| **creator** | 内容、文档 | TODO/DONE |
| **critic** | 审查、质检 | TODO/DONE/T0 |
| **innovator** | 优化、创新 | TODO/DONE |

### 5.2 任务领取规则

```
1. Agent读取TODO.md
2. 选择匹配自身能力的最高优先级任务
3. 在任务前添加 `[执行:Agent名]` 标记
4. 执行任务
5. 完成后删除任务，追加到DONE.md
```

### 5.3 冲突解决

- 多Agent争抢同一任务 → 优先级高的Agent优先
- 任务执行超时（>预期2倍）→ 自动释放，其他Agent可领取
- 死锁检测 → main Agent强制重新分配
