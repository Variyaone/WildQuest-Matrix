# 量化推送项目

## 项目目标
生成完全可信的量化交易推送，包含策略概述、因子说明、回测结果、交易建议和风险提示。

## 项目背景
用户需要每日量化推送，用于辅助投资决策。推送必须基于完整的数据分析流程，确保数据准确、策略有效、风险可控。

## 关键步骤

### 步骤1: 数据更新
- **执行者**: researcher
- **输入**: 市场数据API
- **输出**: 更新后的市场数据
- **依赖**: 无
- **状态**: pending
- **时间**: 预计10分钟

### 步骤2: 因子计算
- **执行者**: researcher
- **输入**: 更新后的市场数据
- **输出**: 有效因子列表
- **依赖**: 步骤1
- **状态**: pending
- **时间**: 预计15分钟

### 步骤3: 策略设计
- **执行者**: architect
- **输入**: 有效因子列表
- **输出**: 策略方案
- **依赖**: 步骤2
- **状态**: pending
- **时间**: 预计20分钟

### 步骤4: 回测验证
- **执行者**: architect
- **输入**: 策略方案
- **输出**: 回测报告
- **依赖**: 步骤3
- **状态**: pending
- **时间**: 预计15分钟

### 步骤5: 推送生成
- **执行者**: creator
- **输入**: 回测报告
- **输出**: 推送内容（Markdown格式）
- **依赖**: 步骤4

### 步骤6: 质量审核
- **执行者**: critic

- **依赖**: 步骤5
- **状态**: pending
- **时间**: 预计10分钟

## 依赖关系
```
步骤1（数据更新） → 步骤2（因子计算） → 步骤3（策略设计） → 步骤4（回测验证） → 步骤5（推送生成） → 步骤6（质量审核）
```

## 质量标准
- [ ] 年化收益率 > 15%
- [ ] 最大回撤 < 20%
- [ ] 夏普比率 > 1.0
- [ ] 信息比率 > 0.5
- [ ] 数据来源可靠
- [ ] 风险提示充分
- [ ] 逻辑清晰易懂
- [ ] 格式规范美观

## 风险提示
- [ ] 风险1: 历史表现不代表未来 - 在推送中明确提示
- [ ] 风险2: 市场环境变化 - 说明策略适用环境
- [ ] 风险3: 数据延迟 - 说明数据更新时间
