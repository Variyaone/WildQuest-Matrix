## 🎯 核心设计决策

### 1. 技术选型

- **语言**: Python 3.11+（量化生态丰富、开发快）
- **框架**: 基于freqtrade架构（成熟、功能完备）
- **API库**: ccxt 4.0+（统一接口、多交易所）
- **数据库**: SQLite（零配置、轻量、安全）
- **Web框架**: Flask（简单易用、快速开发）

### 2. 系统架构

六大核心模块：
1. **数据采集模块** - WebSocket实时行情、K线数据、订单簿
2. **策略引擎** - 策略加载、指标计算、信号生成
3. **订单执行器** - API交互、订单管理、重试机制
4. **风险控制模块** - 仓位管理、止损止盈、熔断机制
5. **回测引擎** - 历史回测、性能评估、参数优化
6. **监控报警模块** - 实时监控、异常检测、通知告警

### 3. 数据流设计

```
OKX WebSocket → 数据采集 → 策略引擎 → 风险控制 → 订单执行 → OKX API
                                            ↓
                                    数据库记录
                                            ↓
                                    监控报警通知
```

### 4. 决策流设计

```
新K线生成 → 策略触发 → 风险检查 → [实盘?] → {人工确认} → 订单执行 → 结果反馈
```

### 5. 初始策略推荐

- **新手首选**: 双均线策略（DualMA）
  - 年化收益: 15-25%
  - 最大回撤: 10-15%
  - 胜率: 45-55%
  - 风险等级: ⭐️ 低风险

- **进阶选择**: 布林带均值回归、RSI超买超卖

### 6. 安全机制

- ✅ 模拟盘先行（OKX Demo Trading）
- ✅ 人工确认（实盘交易必须手动确认）
- ✅ 多层风控（交易前/中/后三层防护）
- ✅ 熔断机制（亏损超10%自动停止）
- ✅ API密钥加密（环境变量存储）
