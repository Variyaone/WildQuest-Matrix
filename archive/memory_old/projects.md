# 项目：A股量化交易系统

> 启动日期：2026-02-25
> 目标：A股量化交易达到全球顶尖水准
> 阶段：系统自主运行与模拟盘验证

## 项目概述

### 核心目标
- 建立系统性量化交易能力，而非寻找"神模型"
- 管理不确定性，而非消除不确定性
- 年化收益 > 30%，夏普比率 > 2.5，最大回撤 < 15%
- 系统稳定运行 > 90天

### 技术栈
- **语言**：Python
- **数据**：AKShare/TUSHARE（当前使用模拟生成数据）
- **存储**：SQLite + Pickle
- **ML框架**：LightGBM
- **监控**：飞书通知 + 可视化看板

## 核心机制

### 1. 组织自主机制
- **任务池**：`.commander/task_pool.json`
- **看板**：`.commander/kanban.md`
- **自动触发器**：`.commander/auto_trigger.py`（每5分钟扫描）
- **失败重试**：最多3次，间隔5分钟

### 2. 回测引擎V2
- **成本模型**：佣金、印花税、冲击成本
- **基准模型**：LightGBM，时间序列交叉验证，准确率74.25%
- **性能指标**：
  - 总收益率: 946.16%
  - 年化收益率: 83.61%
  - 夏普比率: 2.05
  - 最大回撤: -12.56%

### 3. 模拟盘系统
- **模块化设计**：数据/信号/风控/交易分离
- **文件**：`code/paper_trading.py`

### 4. 自动化架构
- **6个Cron任务**：数据更新、健康检查、监控、回测
- **三层架构**：自动化脚本层 → Cron调度层 → Agent监控层
- **资源优化**：Agent调用减少80%，内存峰值降低75%，CPU峰值降低37.5%

## 战略指导

**核心理念转变**（来自老大）：
- **从"寻找圣杯"到"管理不确定性"**
- **建立系统性能力，而非找到"神模型"**
- **组织自主：让组织能够自主、自动、自发地行动和改进**

**系统自动化要求**：
- **将确定性任务脚本化，减少对agent的依赖**
- **优化算力、内存、网络负载**
- **Agent只在异常时介入**
