## ⚠️ 发现的配置错误

### 错误1：引用不存在的脚本
- **问题**：`morning_push`任务引用了`paper_trading_push.py`，但该文件不存在
- **影响**：8:00盘前推送任务无法执行
- **根本原因**：cron配置未同步于实际脚本变更

### 错误2：8:00任务冲突
- **问题**：同一时间有2个推送任务
  - `morning_push` (8:00) - 引用不存在脚本
  - `morning_master_push` (8:00) - 正常任务
- **影响**：可能执行顺序混乱
- **根本原因**：任务冗余未清理

### 错误3：缺少周度因子评估任务
- **问题**：系统有因子动态评估功能，但没有定时任务调用
- **影响**：因子权重无法自动更新
- **建议**：添加周日因子评估任务

## 🔧 核心流程说明

### daily_master.py 完整流程
```
1. 数据更新 → 加载股票数据，检查质量
2. 因子评估 → 计算IC/IR，更新动态权重
3. 股票分类 → 按行业/市值差异化处理
4. 选股决策 → 基于因子得分选择Top 5
5. 回测验证 → 为每只股票生成回测身份证
6. 持仓管理 → 更新持仓状态，止盈止损检测
7. 报告生成 → 生成增强型推送报告
8. 推送发送 → 发送到飞书webhook
```

### 关键输出文件
- `data/portfolio_state.json` - 持仓状态
- `data/factor_dynamic_weights.json` - 动态因子权重
- `data/factor_evaluation_history.json` - 因子评估历史
- `data/selection_result.json` - 选股结果
