# WildQuest Matrix 架构文档

**版本**: 6.5.0  
**更新日期**: 2026-03-28

---

## 1. 系统概述

### 1.1 系统定位

WildQuest Matrix 是 Variya 开发的数据驱动智能投资决策平台，专注于A股市场的量化选股、组合优化和风险管理。系统采用分层架构设计，从数据获取到交易执行形成完整的量化投资管线。

### 1.2 设计原则

1. **分层解耦**: 每层只依赖直接上层，不跨层调用
2. **数据契约**: 所有输出必须符合数据契约规范
3. **统一接口**: 支撑层提供统一的数据存取接口
4. **异常告警**: 异常情况通过支撑层统一告警
5. **前置检查**: 所有模块共用同一套前置检查机制

### 1.3 核心目标

| 指标 | 目标值 |
|------|--------|
| 测试覆盖率 | ≥70% |
| 数据存储 | ≤5GB |
| 文件数量 | 50-80个Python文件 |
| 启动时间 | 减少50% |
| 数据更新效率 | 增量更新，减少80%时间 |
| 数据质量分数 | ≥80分 |

### 2.2 层级职责与依赖关系

| 层级 | 职责 | 依赖上层 | 被下层依赖 |
|------|------|----------|------------|
| **支撑层** | 调度、监控、存储、日志、配置 | 无（独立） | 所有层 |
| **数据层** | 数据获取、清洗、存储 | 支撑层 | 因子层 |
| **因子层** | 因子计算、验证、挖掘 | 数据层、支撑层 | 信号层 |
| **信号层** | 信号生成、过滤、评估 | 因子层、支撑层 | 策略层 |
| **策略层** | 策略设计、选股、优化 | 信号层、支撑层 | 组合优化层 |
| **组合优化层** | 权重优化、中性化、约束 | 策略层、支撑层 | 风控层 |
| **风控层** | 事前/事中/事后风控 | 组合优化层、支撑层 | 执行层 |
| **执行层** | 实盘交易/仿真回测 | 风控层、支撑层 | 分析层 |
| **分析层** | 绩效分析、归因、报告 | 执行层、支撑层 | 用户 |

## 3. 数据流与输出契约

### 3.1 数据传递规范

| 阶段 | 输入 | 输出 | 输出格式 | 传递给 | 数据契约 |
|------|------|------|----------|--------|----------|
| 支撑层 | 配置文件 | 调度任务/存储服务 | - | 所有层 | 统一接口 |
| 数据层 | 外部API | 市场数据 | DataFrame[OHLCV] | 因子层 | H1-H5验证 |
| 因子层 | 市场数据 | 因子值 | DataFrame[stock, factor] | 信号层 | IC/IR验证 |
| 信号层 | 因子值 | 信号强度 | Dict[stock, score] | 策略层 | 胜率验证 |
| 策略层 | 信号强度 | 选股结果 | Dict[stock, selection_score] | 组合优化层 | 回测验证 |
| 组合优化层 | 选股结果 | 目标持仓 | Dict[stock, target_weight] | 风控层 | 权重归一化 |
| 风控层 | 目标持仓 | 审批结果 | List[trade_order] | 执行层 | 风控规则 |
| 执行层(实盘) | 审批结果 | 成交回报 | Dict[trade_result] | 分析层 | 成交确认 |
| 执行层(仿真) | 审批结果 | 回测报告 | Dict[backtest_result] | 分析层 | 撮合记录 |
| 分析层 | 交易结果 | 分析报告 | HTML/PDF | 用户 | 指标完整 |

### 3.2 数据格式规范

```
数据格式规范:
├── DataFrame: 用于批量数据（因子值、行情数据）
├── Dict: 用于映射关系（股票→权重、股票→信号）
├── List: 用于列表数据（交易指令、持仓列表）
└── JSON: 用于配置和元数据
```

### 3.3 存储路径规范

```
存储路径规范:
├── data/master/     # 主数据（持久化）
├── data/cache/      # 缓存数据（临时）
├── data/metadata/   # 元数据索引
├── data/factors/    # 因子数据
├── data/signals/    # 信号数据
├── data/portfolio/  # 组合数据
└── data/reports/    # 报告数据
```

#### 4.1.2 核心组件

**统一数据获取器 (UnifiedDataFetcher)**
- 功能：汇总所有数据源，提供统一的数据获取接口
- 支持数据源：AKShare、BaoStock、yfinance、Ashare
- 特性：缓存机制、故障切换、优先级管理

**增量更新器 (IncrementalUpdater)**
- 功能：智能检测缺失数据并增量更新
- 特性：缺口检测、断点续传、进度持久化

**数据质量检查器 (DataQualityChecker)**
- 功能：实现数据质量管控要求
- 检查项：H1-H5硬性要求 + E1-E5弹性要求

## 5. 每日执行管线

### 5.1 执行流程

```
每日/每周执行流程:

Step 1: 数据更新
   └─ 增量更新市场数据 → 更新元数据索引

Step 2: 因子计算
   └─ 遍历因子库 → 计算所有因子值 → 存入因子缓存
   └─ (可选) AI因子挖掘

Step 3: 信号生成
   └─ 遍历信号库 → 应用因子组合规则 → 生成信号强度
   └─ (可选) AI增强信号

Step 4: 策略执行
   └─ 选择活跃策略 → 应用信号 → 生成选股结果
   └─ (可选) RL策略优化

Step 5: 组合优化
   └─ 应用优化方法 → 计算目标权重 → 中性化处理

Step 6: 风控检查
   └─ 事前风控 → 通过则生成交易指令

Step 7: 交易执行
   └─ 订单生成 → 信号推送（供交易员手动执行）
   └─ (可选) RL执行器优化

Step 8: 绩效分析与报告
   └─ 计算绩效指标 → 生成分析报告 → 推送通知

Step 9: 监控仪表盘
   └─ 更新监控状态 → 检查预警条件 → 触发回测（如需要）
```

### 5.2 管线实现

系统提供两种管线实现：

**标准管线 (`core.daily.__main__`)**
- 传统因子计算和信号生成
- 标准策略执行和组合优化
- 适用于日常运维

**AI增强管线 (`core.daily.ai_pipeline`)**
- 集成AI因子挖掘
- AI增强信号生成
- RL策略优化和执行
- 适用于研究和优化

### 5.3 前置检查流程

```
管线执行流程：

用户请求执行操作（如推送/交易）
    ↓
调用前置检查管理器
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Step 1: 检查数据层                                          │
│ ├── 检查新鲜度：数据是否过期                                 │
│ ├── 检查硬性要求 H1-H5                                      │
│ └── 检查弹性要求 E1-E5                                      │
│     ├── H失败 → 标记UNTRUSTED → 返回Step1修复               │
│     └── 通过 → 继续                                         │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Step 2-6: 检查因子层/信号层/策略层/组合优化层/风控层         │
│ └── 类似流程                                                │
└─────────────────────────────────────────────────────────────┘
    ↓
所有检查通过 → 标记结果为TRUSTED → 允许执行操作
```

## 7. 错误处理

### 7.1 错误分类

```
错误分类体系：

├── 数据错误 (E001-E099)
│   ├── E001: 数据为空
│   ├── E002: 必需字段缺失
│   ├── E003: 时间序列不连续
│   ├── E004: 价格逻辑错误
│   └── E005: 未来数据泄露
│
├── 因子错误 (E100-E199)
│   ├── E101: 因子不存在
│   ├── E102: 因子计算失败
│   └── E103: 因子验证失败
│
├── 信号错误 (E200-E299)
│   ├── E201: 信号不存在
│   └── E202: 信号生成失败
│
├── 策略错误 (E300-E399)
│   ├── E301: 策略不存在
│   └── E302: 策略执行失败
│
├── 组合优化错误 (E400-E499)
│   ├── E401: 优化失败
│   └── E402: 权重不归一
│
├── 风控错误 (E500-E599)
│   ├── E501: 风控检查失败
│   └── E502: 超过权重上限
│
├── 交易错误 (E600-E699)
│   ├── E601: 订单无效
│   └── E602: 资金不足
│
└── 系统错误 (E900-E999)
    ├── E901: 配置文件错误
    └── E902: 网络超时
```

### 7.2 错误恢复策略

| 错误类型 | 恢复策略 | 重试次数 | 回退 |
|----------|----------|----------|------|
| 数据为空 | 检查数据源 | 3次 | 否 |
| 数据过期 | 重新获取数据 | 3次 | 是 |
| 因子计算失败 | 记录日志，跳过 | 1次 | 否 |
| 策略执行失败 | 回退到默认策略 | 1次 | 是 |
| 优化失败 | 使用等权配置 | 0次 | 否 |
| 风控检查失败 | 拦截交易，告警 | 0次 | 是 |
| 网络超时 | 指数退避重试 | 5次 | 否 |
