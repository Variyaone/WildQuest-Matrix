# 心跳工作流

## 锚定（每次必执行）

1. 目标解析 → 任务拆解 → 资源调度 → 进度监控 → 交付验证
2. 读SENTINEL_LOG.json确认模式（normal/maintenance/readonly）
3. **非normal模式立即返回HEARTBEAT_OK**

## 核心流程
| 步骤 | 动作 |
|---|---|
| 输入 | 分析消息，生成任务写TASK_POOL.md |
| 领取 | 按优先级领取 |
| 执行 | 大任务原地分解，审查完成后git提交 |
| 归档 | 更新状态，归档ARCHIVE.md |
| 探索 | 无P1任务时触发延续/回顾/复盘/改进 |

## 核心检查清单
**Sentinel**: SENTINEL_LOG.json | TASK_POOL.md

## T0级要求（2026-03-19）
- 任务完成必须git提交
- 任务原子化分解（<5分钟，<200行）
- 评审官默认怀疑态度，必须实测验证

## 静默时段（2026-03-20添加）
为减少不必要的资源消耗，以下时段仅检查T0紧急任务，不启动探索类任务：
- **晚间静默**: 23:00-08:00 GMT+8
  - 仅检查T0异常（资金、致命故障、不可逆操作）
  - 不启动P1/P2/P3任务的探索
  - 仍执行基本健康检查（Sentinel模式、系统状态）
- **正常时段**: 08:00-23:00 GMT+8
  - 完整心跳检查流程
  - 按优先级领取和执行任务

## 异常速查

| 异常 | 处理 |
|---|---|
| 任务停滞>2h | 重启或换agent |


**间隔**: 30分钟（高峰15分钟）
