# 记忆系统架构说明

## 📁 完整结构

```
.openclaw/workspace/
├── MEMORY.md                          # 📋 长期记忆索引（私聊加载）
│
├── memory/                            # 🧠 记忆分区
│   ├── projects.md                    # 📁 项目文档（OKX交易系统）
│   ├── active-tasks.md                # 🗂️ 活跃任务（人读版）
│   ├── lessons.md                     # 📚 经验教训
│   └── 2026-02-25.md                   # 📝 每日日志（原始记录）
│
└── .commander/                        # ⚙️ 运行时状态
    ├── TASK_STATE.json                # 🔄 任务状态（机器读）
    ├── KANBAN.md                      # 📊 任务看板
    ├── OKX_CONFIG.md                  # 🔐 OKX配置
    └── LOG.md                         # 📜 指挥日志
```

## 🔄 文件职责分离

| 文件 | 类型 | 用途 | 读取方式 |
|------|------|------|----------|
| **MEMORY.md** | 索引 | 长期记忆索引，指向具体文件 | 会话启动加载 |
| **memory/projects.md** | 文档 | 项目上下文（OKX技术栈、策略等） | 会话启动+按需 |
| **memory/active-tasks.md** | 文档 | 活跃任务（人类可读版本） | 会话启动+按需 |
| **memory/lessons.md** | 避坑 | 经验教训、最佳实践 | 按 |
| **memory/YYYY-MM-DD.md** | 日志 | 每日原始操作记录 | 会话启动加载 |
| **TASK_STATE.json** | 状态 | 运行时状态（机器可读JSON） | 程序读取 |
| **KANBAN.md** | 看板 | 管理界面显示 | 人工查看 |
| **OKX_CONFIG.md** | 配置 | API凭证、端点等敏感信息 | 程序读取 |

## 🔁 同步机制

### TASK_STATE.json ↔ memory/active-tasks.md
- **TASK_STATE.json**: 机器可读JSON，程序化更新
- **memory/active-tasks.md**: 人类可读Markdown，手动更新或脚本同步

### memory/projects.md ↔ OKX_CONFIG.md
- **memory/projects.md**: 包含OKX配置信息（技术栈、API端点）
- **OKX_CONFIG.md**: 独立的API凭证文件（安全隔离）
  ```json
  注意：OKX_CONFIG.md 包含敏感凭证，不应直接提交到MEMORY.md的摘要中
  ```

### KANBAN.md ↔ TASK_STATE.json
- **KANBAN.md**: 从TASK_STATE.json生成或手动维护
- **TASK_STATE.json**: 状态来源

## 💡 使用原则

1. **程序化操作**：更新TASK_STATE.json（JSON）
2. **人类阅读**：更新memory/*.md（Markdown）
3. **定期同步**：心跳时检查两者一致性
4. **避免重复**：每个职责只有一个文件负责
